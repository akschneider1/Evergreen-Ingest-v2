  • Clean before you index. The drift report shows exactly which content is stale. Remove or correct those passages in your source documents before chunking and embedding — a semantically similar query will surface the wrong value just as confidently as the right one. Before embedding each confirmed chunk, prepend its validated attributes as plain text (e.g. Parameter: filing threshold | Value: $14,600 | Source: policy) to give the embedding model more signal, then store those fields as metadata for filtered retrieval at query time.
  • Turn drift findings into Evergreen Evals tests. Every drifted or missing parameter gives you a known-correct value (policy), a known-wrong value (stale implementation), and source text for both — enough for a concrete hallucination test. Index both versions, ask the question, assert the model answers with the policy value. SME-confirmed decisions become labelled ground truth. Run the suite after any KB update to catch regressions before they reach users.
